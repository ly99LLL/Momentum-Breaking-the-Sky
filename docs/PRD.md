## 1. 硬件约束与设计回应

### 1.1 三大约束及其设计后果

这是整个PRD的根基。每一个设计决策都从这三个约束出发。

| # | 约束 | 具体参数 | 设计后果 |
|---|------|---------|---------|
| **1** | **笔记本内置摄像头** | 位置：屏幕上方边框，不可移动<br>分辨率：720p（常见）或1080p（较新机型）<br>FOV：约60-78°（较窄）<br>低光性能：差（小传感器，高噪声）<br>帧率：30fps（多数）或60fps（少数） | 用户与屏幕距离=摄像头距离（40-60cm）。摄像头只能拍到面部+肩+上臂。双手向两侧大幅伸展时可能出画。**交互必须在摄像头FOV的"安全区"内设计。** 低光性能差→需要提醒用户开灯。 |
| **2** | **笔记本内置扬声器** | 低频截止：通常150-200Hz以下无声<br>立体声分离度：差（两扬声器间距约30cm）<br>最大音量：有限（通常2-5W）<br>音质：中频为主，高频有但单薄 | **没有次声波。没有震撼低频。没有空间音频。** 声音设计必须完全依赖中频和高频来传达情感。优势：用户离扬声器近（50cm），细节可闻。 |
| **3** | **RTX 5070 笔记本GPU** | 算力：约25-30 TFLOPS FP32<br>显存：8-12GB GDDR7<br>功耗墙：笔记本散热限制，长时间满载可能降频 | 桌面版的渲染性能预算需要预留散热余量。自适应渲染更加重要——不能假设GPU永远满血。 |

### 1.2 笔记本摄像头FOV的精确分析

```
摄像头FOV: 70°（典型笔记本内置摄像头）
用户距离屏幕: 50cm
摄像头位置: 屏幕上方，略高于眼睛，约向下5-10°俯拍

在50cm距离处，70° FOV的可见范围：
- 水平: 约70cm宽
- 垂直: 约50cm高（16:9画幅）

用户身体在画面中的范围（坐姿，距屏幕50cm）：
- 面部: 画面中央偏上 ← 始终可见，高置信度
- 双肩: 画面中下部 ← 通常可见
- 双肘: 画面下边缘 ← 可能被裁切
- 双手（自然放在身前）: 画面下边缘附近 ← 依赖坐姿
- 双手（向两侧伸展）: 超出画面 ← 不可追踪
- 双手（向上举）: 超出画面 ← 不可追踪
- 腹部以下: 不可见 ← 不在FOV内

结论：交互设计只能依赖 面部 + 肩部 + 肘部 + 手腕（在安全区内）
```

### 1.3 笔记本扬声器的频率响应分析

```
典型笔记本扬声器频率响应：
- 20-150Hz:  ✗ 完全无声（物理限制，振膜太小）
- 150-300Hz: △ 极弱（能听到一点但没力量）
- 300-800Hz: ✓ 可闻（中低频基础）
- 800-3000Hz: ✓✓ 最好（中频，人声和主要乐器区）
- 3000-8000Hz: ✓ 可闻（高频，但可能尖锐）
- 8000Hz+: △ 衰减（超高频不可靠）

声音设计策略：
❌ 删除：次声波层（20-40Hz）—— 完全听不到
❌ 删除：低频冲击（地震感）—— 振膜无法还原
❌ 删除：空间音频（HRTF）—— 笔记本扬声器物理上无法产生空间感
✅ 保留：中频纹理、泛音、节奏变化
✅ 新增：立体声节奏脉动（利用左右扬声器分离，虽然窄但是有）
✅ 新增：频率上移——将原"低频压迫"的感觉用中频不和谐音程替代
```

### 2.1 一句话定义

> 一个双击即开的.exe桌面应用。打开笔记本摄像头，在屏幕前用双手蓄力、撕裂数学构建的混沌空间。每次体验生成一张独特的数字狂草拓片。

```
硬件要求：
├── GPU: NVIDIA RTX 5070 笔记本GPU（必需）
├── 摄像头: 笔记本内置摄像头（必需，720p+即可）
├── 扬声器: 笔记本内置扬声器（必需）
├── 内存: 16GB+（推荐）
└── 存储: 100MB 空闲空间

├── 操作系统: Windows 11（主要支持）/ Windows 10，（可以用python环境以及库）

应用形态：
└── 单个.exe文件（Tauri打包，约35MB）

    ├── 自动全屏
    ├── 请求摄像头权限（首次）
    └── 即可体验
```

### 2.4 核心创新点

| # | 创新点 | 说明 |
|---|--------|------|
| 1 | **笔记本前置摄像头交互范式** | 在极其受限的硬件条件下（固定视角、窄FOV、低分辨率）实现富有表现力的身体交互——只用面部+肩部+手腕关键点 |
| 2 | **势能累积触发机制** | 不是挥手触发，不是按键触发——必须"先蓄后爆"，蓄势时间和爆发强度都达标才会撕裂空间 |
| 3 | **不可逆SDF拓扑撕裂** | Raymarching + SDF布尔运算——撕裂不是播放动画，是底层数学函数的真实改变 |
| 4 | **约束性撕裂（狂草法度）** | 撕裂路径被吸附到空间的内在"脆弱面"，对应狂草书法"法度中的自由" |
| 5 | **✨速度-飞白连续映射** | 手腕速度实时映射为笔触质感：慢速=实线饱满，快速=飞白枯笔，极速=墨迹溅射。书法中"徐疾相生"的数字化 |
| 6 | **✨意在笔先（意图预判）** | 蓄势阶段系统预判撕裂方向，在空间中浮现淡光痕——对应书法"意在笔先，然后作字" |
| 7 | **✨空中书写轨迹** | 撕裂后用户可在空间中持续"书写"，手腕轨迹化为金色狂草笔画——体验从"触发动画"升级为"身体书写" |
| 8 | **生成式数字狂草拓片** | 每次体验生成一张独特的4K印记（含撕裂路径+书写轨迹），自动保存，可导出分享 |
| 9 | **中频驱动的笔记本声音设计** | 完全放弃次声波和低频冲击，用中频不和谐音程、节奏变化、静默对比来传达压迫感和释放感 |

## 3. 文化价值体系

### 3.1 核心文化叙事（笔记本版）

笔记本是一个"数字窗口"。每天早上打开它，面对的就是信息的混沌——邮件、消息、推荐流、待办事项。这件作品把这个日常体验变成了一个仪式：

> **双手撕开混沌 = 用意志力在信息洪流中开辟秩序。**

这不是"盘古开天"的宏大叙事，而是**每个普通人每天早上都在做的事**——只是这次，你的身体真的在改变一个数字空间。

### 3.2 为什么是狂草？（五体选择论）

书法五体：篆、隶、楷、行、草。草书中又分章草、今草、狂草。

| 书体 | 特征 | 为什么不用 |
|------|------|-----------|
| 篆书 | 圆润匀称，静态对称 | 太安静——没有"撕裂"的身体张力 |
| 隶书 | 蚕头燕尾，装饰性强 | 太规整——笔画是"设计出来的"不是"爆发出来的" |
| 楷书 | 端正规范，一笔一画 | 太理性——每个点画都是深思熟虑的 |
| 行书 | 流畅连贯，实用性强 | 接近了，但仍然是"写给别人看的"——有社交自觉 |
| **狂草** | **极度自由、全身书写、情绪宣泄** | **✅ 唯一选择——不是"写字"，是"身体在纸上的投影"** |

**狂草不是"最潦草的字体"，而是"最身体的艺术形式"。** 张旭观公孙大娘舞剑器而悟笔法——说明狂草的本质不是二维的，是身体在三维空间中的运动被压缩到二维纸面。我们的交互恰好是这个过程的**逆运算**——把二维书法展开回三维身体运动。

### 3.3 狂草五大美学原则 → 交互映射

这是本次PRD修订的核心——确保**每条交互都在书法中有根**。

| 美学原则 | 书法中的含义 | 数字交互映射 | 检测方式 |
|---------|-------------|------------|---------|
| **一气呵成** | 整幅作品从第一笔到最后一笔，气息不断 | 体验从"困局"到"收锋"是一个完整的气息弧线，中间不中断 | 状态机全程连接，无硬切 |
| **徐疾相生** | 运笔有时如奔雷、有时如抽丝，速度对比产生节奏 | ✨ 手腕速度连续映射为笔触质感：疾=飞白枯笔，徐=饱满实线 | `wristSpeed` → 笔触纹理参数 |
| **意在笔先** | 落笔之前，心中已有笔画的形态和方向 | ✨ 蓄势阶段系统预判撕裂方向，空间浮现淡光痕 | 肩肘角度趋势 → 预判方向向量 |
| **飞白见性** | 枯笔快速擦过纸面，露出底色的纹理——最能见书家的真性情 | ✨ 高速移动的轨迹自动产生飞白效果，速度越快飞白越明显 | `wristSpeed > V_threshold` → 飞白混合比 |
| **顿挫有节** | 不是一直滑过去——有停顿（顿）和方向突变（挫），产生节奏 | ✨ "顿"=速度骤降至零，轨迹局部变粗；"挫"=方向突变的折角 | 速度极小值检测 + 方向角变化检测 |

### 3.4 狂草书法的身体映射（扩展版）

狂草书法家（张旭、怀素）创作时的身体状态：站立、凝神、蓄势、爆发、收锋。这不是"用手指写字"，而是**用整个身体的力量通过毛笔传递到纸上**。

**全身书写的五层发力分解：**

| 身体部位 | 书法中的作用 | 笔记本版的简化映射 | 追踪状态 |
|---------|------------|------------------|---------|
| 足（站桩） | 重心下沉，稳定根基——全身力量的起点 | 坐姿/站姿选择。站立=更大的动作空间，坐姿=稳定但幅度受限 | 间接（通过身体Z轴稳定性推断） |
| 腰（发力） | 转腰带动整条手臂——大幅笔画的动力源 | 躯干前倾/后仰 = 力量积蓄/释放。前倾=蓄势加速 | MediaPipe 肩部Z轴变化 |
| 肩（运臂） | 决定笔画的大方向和格局 | 肩肘角度变化 = 撕裂方向和大势 | MediaPipe 肩-肘连线角度 |
| 肘（转腕） | 中距离的笔画转折 | 肘部外展/内收 = 撕裂的力度和广度 | MediaPipe 肘部位置 |
| 腕（执笔） | 精细的笔画细节——速度、轻重、飞白 | ✨ 手腕运动 = 书写轨迹的质感（飞白/实线）和节奏（顿挫） | MediaPipe 手腕位置（置信度依赖） |

**核心理念：书法不是手指的运动，是全身的运动。同样，撕裂不是手腕的抖动，是蓄势后的全身爆发。书写不是手指在空气中比划，是"足→腰→肩→肘→腕"完整发力链的末端呈现。**

**书法的完整身体过程在本作品中的映射：**

| 书法的身体过程 | 本作品的交互映射 | 追踪方式 |
|---------|---------------------|
| 书法家站立于案前 | 用户坐/立于笔记本前 | — |
| 凝视宣纸，屏息凝神 | 凝视屏幕，混沌在呼吸（呼吸节奏可与之锁定） | 肩部Z轴微变 → 呼吸频率 |
| 蘸墨、调锋（蓄势） | 双手合拢，身体前倾——势能累积 | 肩肘角+手腕速度+身体前倾 |
| **意在笔先**——落笔前心中已有笔画 | 系统预判撕裂方向，空间浮现淡光痕 | 肩肘角度趋势+手腕运动趋势 |
| 落笔——力量从肩到臂到腕到笔尖 | 撕裂——三种方向（横撕/纵斩/推破）产生不同裂缝拓扑 | 肩肘角突变+手腕速度峰值 |
| 笔走龙蛇，一气呵成 | **空中书写**——手腕轨迹化为金色狂草笔画 | 手腕在FOV内连续移动 |
| 疾徐轻重，节奏变化 | **速度→飞白映射**——快速=枯笔，慢速=实线 | 手腕速度连续映射笔触参数 |
| 顿挫转折，笔断意连 | **顿挫检测**——停顿=墨韵晕染，折角=墨迹溅射 | 速度极小值+方向突变检测 |
| 蘸墨再书，层层叠加 | **复笔循环**——可多次蓄力→撕裂（最多3次），新裂缝与旧裂缝交叉 | 再次检测合拢+前倾+爆发序列 |
| 收锋回锋，气息渐平 | **主动收锋**——双手回收速度控制轨迹消散速度 | 手腕速度<10cm/s持续>2s |
| 作品完成，一方印章 | 印记生成——含撕裂路径+书写轨迹+题跋 | 最高品质渲染+笔触叠加 |

> 这11个环节覆盖了狂草书法创作的完整身体过程。任何一个环节的缺失都会让体验不完整——就像一幅狂草少了收锋会气息涣散，少了蓄势就没有爆发力。

### 3.5 三层文化编码

| 层次 | 内容 | 笔记本版传达 | 依赖知识 |
|------|------|------------|---------|

### 5.2 学术贡献

1. **"笔记本摄像头交互"的极限探索：** 在最低配的传感条件下（固定720p摄像头、无深度、窄FOV），通过精心的交互设计和算法（势能累积+卡尔曼滤波+脆弱面引导），实现了富有表现力的身体交互。这为"人人可用的身体交互"提供了参考——不需要Kinect，不需要RealSense，只需要笔记本自带的摄像头。

2. **"势能累积"作为通用交互原语：** 这是一个新型的离散触发机制。与"点击""长按""手势识别"不同，"势能累积"要求：(a)持续的能量积累期 + (b)瞬时爆发 + (c)积累不够则爆发无效。这可以推广到需要"防止误触发的重要操作"的场景。

3. **"约束性自由"设计原则：** 从狂草书法提炼——系统提供结构（脆弱面/张量场），用户提供时机和力度。这种"半约束"的交互在创造力和可控性之间找到了平衡点。

4. ✨ **"速度-飞白"连续映射作为身体表达通道：** 不同于现有的手势识别（离散分类），本作品将手腕速度连续映射为视觉笔触质感——慢速=饱满笔画，快速=枯笔飞白。这为"身体数据→视觉质感"的连续映射提供了一个可推广的模型：不把身体动作分类为几个手势，而是让动作的细微差异直接可见于视觉输出。

5. ✨ **"意在笔先"——人机意图共谋：** 在用户完成动作之前，系统已预判其意图方向并以淡光痕可视化。这不是传统的"预测用户行为以加速响应"，而是"将预判可视化以引导用户"——系统不是被动的执行者，而是与用户共同完成创作的协作者。

6. ✨ **狂草书法作为交互设计的文化操作系统：** 本文提出了一套方法论——从特定文化实践（狂草五大美学原则）出发，逐条推导交互设计决策，并用"对账表"确保每条交互都能在文化中找到对应。这种"文化→交互"的推导方法可为其他文化驱动的交互设计提供参考。

**MediaPipe Pose 在笔记本距离（40-60cm）下可追踪的关键点：**

```
高置信度（几乎总是可用）：
├── 鼻子 (0)           — 头部位置，始终在画面中
├── 左眼/右眼 (1-6)    — 头部朝向
├── 左肩/右肩 (11,12)  — 🌟 核心指标

不可用：
└── 膝盖、脚踝等       — 完全出画，不追踪
```

**笔记本版势能指标（只用可用关键点）：**

```javascript
// 三个核心指标，全部基于笔记本摄像头FOV内的关键点

// 指标1: 肩肘角 — 上臂与躯干的夹角
// 蓄势时: 肘部靠近身体，角度缩小
// 撕裂时: 肘部外展，角度突然增大
shoulderElbowAngle = angle(shoulder, elbow, hip_estimated)

// 指标2: 手腕可见区的运动速度
// （手腕在画面内才能追踪到，出画时使用肘部位置作为代理）
wristSpeed = |wristPosition(t) - wristPosition(t-1)| / dt
// 如果手腕置信度 < 0.4，退化为: wristSpeed = elbowSpeed × 1.5

// 指标3: 身体前倾度 — 肩部Z轴位置的变化
// 笔记本摄像头的Z轴不精确但方向是对的——靠近/远离可区分
leanForward = shoulderZ_avg - baselineZ
// 前倾(蓄势) = 正值 | 后仰(释放) = 负值

// 综合势能
E(t) = ∫[t-T, t] w(τ) × [
  0.4 × ΔshoulderElbowAngle(τ) +  // 肘部收拢→外展
  0.35 × wristSpeed(τ) +           // 手部运动强度
  0.25 × leanForward(τ)            // 身体前倾
] × confidencePenalty(τ) dτ

// confidencePenalty: 如果关键点置信度<0.4，该帧权重降为0.3
```

### 6.4 渲染管线（笔记本GPU适配）

**关键约束：** 笔记本RTX 5070受散热限制，长时间满载会降频。需要预留散热余量。

```
自适应渲染（笔记本版）：

屏幕分为3个环形区域：

区域1：中央圆 (半径 = min(w,h) × 0.3)
  → 全分辨率
  → 64 raymarch步
  → 3层domain warping

区域2：中间环 (0.3 → 0.7)
  → 半分辨率（2×2像素块）
  → 32 raymarch步
  → 2层domain warping

区域3：边缘 (0.7 → 1.0)
  → 1/4分辨率（4×4像素块）
  → 16 raymarch步
  → 1层domain warping

过渡：区域之间2-4px高斯模糊

笔记本性能预算（1920×1080全屏）：
- 等效计算量: ~65M raymarch步/帧（自适应后）
- RTX 5070笔记本GPU: ~8ms/帧 → 125fps理论值
- 实际目标: 60fps（预留散热余量，不推满GPU）

散热策略：
- 目标GPU利用率: 70-80%（不是100%）
- 如果GPU温度 > 82°C 持续5秒 → 自动将区域2降为1/4分辨率
- 如果GPU温度 > 88°C → 强制区域1也降为半分辨率
- 用户几乎注意不到视觉差异（自适应渲染+高斯平滑）
```

| ✨ **书写·走笔** | 金色轨迹 `#D4A574` + 飞白纹理 `#C5B89E`(枯) + 背景青白 `#E8ECEF` | 金色狂草笔触在青白背景上流动，飞白处露出"纸底"纹理。笔触核心浓度与手腕速度反相关 | 金色在笔记本屏幕上辨识度高；飞白的低饱和度区需要IPS面板区分 |
| ✨ **收锋·回势** | `#E8ECEF` → `#C4BFB5`(暖灰) + 金色淡出 `#8B7D6B` | 青白渐变为暖灰，金色轨迹随回收速度淡化至消失。整体色调从"冷"转"暖"——像宣纸在黄昏光下 | 暖灰色在任意屏幕上表现稳定 |
| **清明·气韵** | `#E8ECEF`(天青) + `#8B7D6B`(赭墨) | 通透、流动、低饱和 | 低饱和色在笔记本屏幕上准确度较高 |

**最小化保证：** 即使在最差的笔记本屏幕（TN面板、60% sRGB）上，三个阶段的亮度差异和色温差异仍足以区分——因为编码了亮度和纹理密度双重信息。

### 7.2 屏幕作为"画窗"

笔记本屏幕约13-16英寸，距离眼睛40-60cm。视觉隐喻：**屏幕是一扇小窗，窗那边是混沌的世界。**

```
设计原则：
1. 混沌始终在"窗户那边"——屏幕内是另一个世界
2. 屏幕四边保持微弱暗角（<8%），暗示"窗框"
3. 撕裂时，裂缝的光溢出到屏幕边缘——暗示"世界比窗户更大"
4. 全屏独占模式——没有标题栏、没有任务栏、没有任何UI chrome
5. 鼠标光标3秒不动自动隐藏
6. 屏幕反光中用户隐约看到自己的脸——这不是bug，增强了"我在窗外看窗内"的对面感
```

### 7.3 印记的视觉设计

印记不是截屏。它是专门"摆拍"的视觉产物。体验结束后，用最高品质渲染一帧（无自适应降质，全64步raymarch，3层warping），然后叠加狂草风格的撕裂路径。

```
印记生成流程（v31扩展）：
1. 清明阶段 → 选择最具"画面感"的一帧（优先选择包含书写轨迹+裂缝的帧）
2. 用最高品质重新渲染该帧（4800×6400内部分辨率 → 下采样到2400×3200）
3. 将体验者的撕裂路径以"狂草笔触"风格叠加
   - 路径核心 = 浓墨（极暗，但保留纹理）
   - 路径边缘 = 飞白效果（高频噪声残留在裂缝边缘）
   - 金色轨迹 = 保留原色（不叠加墨色）
   - ✨ 书写轨迹 = 以金色狂草笔画叠加在裂缝之上（如碑刻般嵌入空间）
     - 轨迹的速度信息保留为飞白密度——快速段飞白多，慢速段笔画饱满
     - 顿挫点的墨韵晕染保留
     - 挫笔的溅射粒子保留（随机采样方向）
4. 题跋生成（竖排，繁体，右下角）：
   [年月日] 於 [城市] · 第[N]度開天
   撕裂深度 [Lv.X]
   筆數 [獨筆/雙飛/三疊] ← ✨新增
   風格 [剛猛/流暢/凝澀]
   ✨ 頓挫 [N]處 · 飛白 [M]筆 ← ✨新增
5. 整体覆盖极淡的"宣纸"纹理（噪声强度<3%）
6. 输出PNG
```

## 8. 交互体验设计

### 8.1 笔记本身体语言词典（v31扩展版）

所有动作在笔记本摄像头FOV内可行。坐姿或站立均可。推荐站立（动作空间更大），但坐姿也能完整体验。

#### 8.1.1 基础动作（v30保留）

| 动作 | 检测指标 | 含义 | 效果 |
|------|---------|------|------|
| **试探** | 单手小幅移动，速度<15cm/s | "这是什么？" | 混沌产生微弱涟漪 |
| **合拢** | 双肘向身体收拢，肩肘角缩小，持续>1s | 蓄势——"聚气" | 混沌向屏幕中心坍缩，颜色变暖 |
| **前倾** | 肩部Z轴向摄像头靠近 | 专注——"凝视深渊" | 势能累积加速 |
| **撕裂·横** | 双肘突然外展，肩肘角瞬间增大 | 爆发——"撕！" | 水平裂缝 |
| **撕裂·纵** | 双手（腕）从高处向低处快速移动 | 爆发——"斩！" | 纵向裂缝 |
| **撕裂·推** | 身体瞬间后仰 + 双手前推 | 爆发——"破！" | 中心爆裂 |
| **静止** | 所有关键点速度<5cm/s，持续>3s | 犹豫/欣赏 | 如未撕裂：压迫增强。如已撕裂：轨迹消散加速 |
| **分心** | 头部旋转>25°偏离屏幕 | 注意力转移 | 势能累积暂停 |

#### 8.1.2 ✨ 书写动作（v31新增——撕裂后可用）

> 触发条件：撕裂已发生，空间处于"清明"状态。这些动作用手腕轨迹在空间中"写"出金色狂草笔画。

| 动作 | 检测指标 | 书法对应 | 效果 |
|------|---------|---------|------|
| ✨ **空中书写** | 单/双手腕在摄像头FOV内持续移动（任意方向），速度>3cm/s，持续>0.5s | **运笔走龙蛇**——连续的空间轨迹 | 手腕当前位置生成金色狂草笔触，跟随手腕运动 |
| ✨ **疾书** | 手腕速度 > 40cm/s | **快速运笔/飞白**——枯笔擦过纸面 | 轨迹边缘出现飞白纹理（噪声密度与速度正相关），核心仍为金色但饱和度降低 |
| ✨ **徐行** | 手腕速度 5-20cm/s，方向稳定 | **中锋慢运**——力透纸背 | 轨迹饱满厚实，金色浓度最高，笔画边缘光滑 |
| ✨ **顿** | 手腕速度从>15cm/s骤降至<3cm/s，且停顿>0.4s | **顿笔**——力聚一点 | 轨迹在停顿处局部变粗（半径×1.5-2.0）+ 短暂金光脉冲 |
| ✨ **挫** | 手腕方向在<0.3s内变化>60°（方向突变） | **挫笔**——急转折 | 轨迹出现折角 + 小范围"墨迹溅射"粒子（10-20个暗金粒子向折角外侧飞散） |
| ✨ **收锋** | 撕裂后，双手从展开位置缓缓收回至身体（速度<10cm/s，持续>2s） | **回锋收笔**——笔收回锋 | 金色轨迹随回收速度逐渐消散：快收→快散，慢收→慢散。空间颜色从青白渐变为暖灰 |
| ✨ **复笔** | 在"书写·走笔"或"清明"阶段，再次执行"合拢→前倾→爆发"序列 | **蘸墨再书**——多次蘸墨，层层叠加 | 触发第二次（或第三次）撕裂，新裂缝与已有裂缝交叉，生成更复杂的复合画面 |

**三种撕裂方向产生不同形态的裂缝：**
- 横撕（水平）→ 对应书法"横画"——开阔、延展
- 纵斩（垂直）→ 对应书法"竖画"——锐利、纵深感
- 推破（前后）→ 对应书法"点"的爆发——辐射状裂纹

### 8.2 ✨ 连续交互维度（v31新增）

> v30的交互是离散的——"蓄够了→触发→看动画"。v31的核心升级是把多个交互维度变为**连续的**——用户的每个动作细节都直接可见于视觉输出。

#### 8.2.1 速度-飞白连续映射

这是v31最重要的交互创新。手腕速度不再是"快/慢"的二元判断，而是连续映射到笔触质感。

```
速度→笔触映射函数：

设 v = 手腕瞬时速度（cm/s），v ∈ [0, Vmax]
归一化: s = clamp(v / Vmax, 0, 1)

笔触参数：
├── 核心浓度 C(s) = 1 - s²
│   慢速(s→0): 金色浓度=100%（实笔，饱满）
│   快速(s→1): 金色浓度=0%（纯飞白/枯笔）
│
├── 飞白噪声密度 N(s) = s²
│   慢速: 噪声密度≈0（笔触边缘光滑）
│   快速: 噪声密度≈1（边缘布满飞白纹理）
│
├── 笔触宽度 W(s) = W_base × (1 - 0.4s)
│   慢速: 标准宽度（中锋用笔，笔画饱满）
│   快速: 宽度缩减到60%（枯笔，笔画变细）
│
└── 飞白粒子 P(s) = int(20 × s)
    慢速: 0个粒子
    极速: 最多20个墨点粒子从轨迹边缘飞散

视觉结果：
- 慢速书写 → "颜真卿式"饱满笔画
- 中速书写 → 带轻微飞白的流畅笔画
- 快速书写 → "张旭式"狂放枯笔，飞白满纸
- 极速书写 → 笔触几乎消失，只剩飞白纹理和溅射粒子
```

**技术实现：** 手腕速度经过卡尔曼滤波去抖（τ=0.15s），然后驱动笔触shader中的噪声混合参数。不是切换——是每个像素的混合比都由速度决定。

#### 8.2.2 顿挫节奏检测

```
"顿"的检测：
  IF wristSpeed(t-0.2) > 15cm/s AND wristSpeed(t) < 3cm/s:
    → 触发"顿"事件
    → 当前位置的笔触半径 × 1.8（持续0.5s后回缩）
    → 触发短促共鸣音（木鱼质感，600Hz，持续150ms）
    → 产生轻微金色脉冲光环（半径5→30px，1s内淡出）

"挫"的检测：
  IF directionChange(t-0.3, t) > 60°:
    → 触发"挫"事件
    → 折角处笔触断裂0.1s（模拟笔锋离纸）
    → 折角外侧生成8-15个暗金粒子，以折角方向飞散（速度50-150px/s，寿命0.5-1.5s）
    → 触发墨溅音效（800-3000Hz短噪声，持续80ms）

节奏分析（不强制触发，但影响印记的"风格"判定）：
  统计整个书写阶段的"顿"次数和"挫"次数
  → 顿挫频繁(5+次): 风格="凝涩"（怀素式，节奏鲜明）
  → 顿挫适中(2-4次): 风格="流畅"（张旭式，一气呵成）
  → 顿挫稀少(0-1次): 风格="刚猛"（力量型，不计小节）
```

#### 8.2.3 意在笔先（意图预判）

```
蓄势阶段（撕裂触发前），系统持续追踪用户的"意图方向"：

意图方向计算：
  D_intent(t) = normalize(
    0.4 × D_shoulderElbow(t) +   // 肩肘张开方向
    0.35 × D_wristTrend(t) +      // 手腕最近0.5s的运动趋势
    0.25 × D_lean(t)              // 身体前倾方向
  )
  其中 D_* 均为2D方向向量（在摄像头图像平面内）

预判可视化：
  当势能 E(t) > 阈值的60% 时：
    空间中沿 D_intent 方向浮现极淡的光痕（亮度 < 5% 背景亮度）
    光痕长度 = 30 × (E(t) / E_threshold) px
    光痕颜色 = #8B7355（淡赭金色），透明度随势能上升
    光痕形态 = 沿脆弱面约束后的路径（不是直线）

  当势能 E(t) > 阈值的85% 时：
    光痕亮度提升至 12%
    光痕两端出现微弱的"起笔点"和"收笔点"光斑

文化依据：
  王羲之《题卫夫人笔阵图后》："夫欲书者，先乾研墨，凝神静思，预想字形大小、
  偃仰平直、振动，令筋脉相连，意在笔前，然后作字。"
  ——"意在笔前"的数字化：系统在用户"落笔"（撕裂）前，已预判笔画（裂缝）的形态。
```

#### 8.2.4 呼吸同步（气沉丹田）

```
呼吸近似检测（笔记本摄像头方案）：
  肩部Z轴的微变频率 ≈ 呼吸频率
  → 对 shoulderZ(t) 做带通滤波（0.1-0.5Hz，对应6-30次/分钟）
  → 提取主频率 f_breath_user

混沌呼吸频率：
  混沌的"呼吸"（层1音频的LFO频率）有一个基础频率 f_breath_chaos
  初始值: 0.12Hz (5秒吸 / 7秒呼)
  蓄势阶段: 0.25Hz (2秒吸 / 2秒呼)

呼吸锁定检测：
  IF |f_breath_user - f_breath_chaos| < 0.03Hz 持续 > 8秒：
    → "呼吸同步"状态激活
    → 势能累积速率 × 1.4
    → 屏幕四角出现微弱的暖色呼吸光晕（与呼吸同频）
    → 触发提示音：低频风铃声（523Hz，持续2s，音量极低，接近可闻阈值）
    → 解锁成就："气沉丹田"（静默成就，不弹窗，仅记录在 stats.json）

  这个机制是"可选的深度层"——不同步也能完整体验，同步了有加成。
  大部分用户第一次体验不会触发（需要8秒持续同步），
  多次使用后可能自然发现——这正是书法"功夫在诗外"的体现。
```

复笔循环逻辑：

### 8.3 ✨ 交互设计原则（v31新增）

> 从狂草美学提炼出的交互设计原则。这不是"设计系统的价值观"——这是从书法出发的操作性约束。

| 原则 | 来源 | 交互约束 |
|------|------|---------|
| **身体直接性** | 书法是全身运动 | 每个交互动作的视觉反馈延迟 < 100ms。用户不需要"等待系统处理" |
| **速度可见性** | 徐疾相生 | 手腕速度的差异必须肉眼可见于笔触质感——不是参数，是事实 |
| **方向预判性** | 意在笔先 | 系统必须提前显示用户即将产生的效果（淡光痕），不能"事后才显示" |
| **损伤不可逆** | 落笔无悔 | 裂缝一旦生成不可撤销。书写轨迹一旦写下不可擦除。没有"撤销"按钮 |
| **收束完整性** | 无往不收 | 每次体验必须有明确的"收锋"动作来完成——不能突兀结束或自动跳转 |
| **层次叠加性** | 多次蘸墨 | 复笔产生的画面比单笔更丰富——系统鼓励但不强制复笔 |

### 8.4 隐性引导（v31扩展版，从7层→9层）

| 层 | 机制 | 笔记本版实现 |
|---|------|------------|
| 1 | **空间响应** | 手动→涟漪。最直接、最即时。用户30秒内建立"我=因，空间=果"的认知。 |
| 2 | **渐进压迫** | 如果用户只是小幅试探，60秒后混沌变得更稠密、更暗——不适感驱使用户做更大的动作。 |
| 3 | **坍缩中心** | 屏幕中央出现微弱暖色光点——人的视觉注意力本能被暖色吸引。双手自然向中心靠拢。 |
| 4 | **中心引力** | 双手距离坍缩中心越近，涟漪越强、周围越亮——正反馈引导双手向中心集中。 |
| 5 | **临界暗示** | 势能接近阈值时：屏幕四角出现极微弱的光裂纹（<5%亮度）+ 嗡鸣音高微微颤抖。 |
| 6 | **爆发释放** | 撕裂触发：0.3s完全静默 → 巨大视听爆发。对比越强烈，释放感越强。 |
| 7 | **余韵** | 撕裂后空间不再主动回应用户动作。用户自然从"做"切换为"看"。 |
| 8 | ✨ **速度示范** | 如果用户在"书写·走笔"阶段只用单一速度书写超过60秒，清明空间边缘出现两个淡影（约8%亮度）：一个快速划过（演示"疾书/飞白"），一个缓慢移动（演示"徐行/实笔"）。3秒演示后消失。不重复播放。 |

- 系统温和介入
- 混沌中出现两个极淡的光斑（在理想的双手位置）

## 9. 声音设计（笔记本扬声器版）

### 9.1 笔记本扬声器的物理限制与设计应对

这是整个声音设计的起点：

| 限制 | 应对策略 |
|------|---------|
| **150Hz以下完全无声** | 删除所有次声波层和低频冲击。用中频不和谐音程（增四度、小二度）来制造"压迫感"——这些音程本身的紧张感不依赖低频。 |
| **立体声分离度差** | 不做空间音频。改为利用左右声道的**节奏交替**——左右声道以略有相位差的方式交替脉动，制造"空间在呼吸"的感觉。 |
| **音量有限** | 依赖**动态对比**而非绝对音量。0.3秒的完全静默→然后中高频爆发——大音量和小音量之间的跳跃比持续的大音量更有冲击力。 |
| **中频温暖** | 笔记本扬声器在中频(300-3000Hz)表现最好。将核心声音设计集中在这个频段。 |

### 9.2 五层声音架构

```
所有频率设计在 180Hz - 10000Hz 范围内（笔记本扬声器可用频段）

层1: 呼吸层（始终存在）
  类型: 带通滤波白噪声（中心频率 400Hz, Q=2）
  调制: 音量以5-8秒周期缓慢起伏（"混沌呼吸"）
  阶段变化:
    初始: 缓慢沉重 (5秒渐强 / 7秒渐弱)
    蓄势: 加速变浅 (2秒/2秒)
    撕裂瞬间: 暂停(0.3s)
    清明: 恢复，但更轻盈 (6秒/6秒)
  实现: 一个LFO控制GainNode

层2: 混沌纹理层
  类型: 粒状合成(granular) — 短小的噪声粒子(50-200ms)随机散布
  频率: 粒子密度 = 10-30个/秒（受空间"密度"参数调制）
  音色: 暗色噪声 — 带通滤波 300-2000Hz
  立体声: 粒子在左右声道随机分布
  调制: 手部运动→粒子密度增加→空间"活性"增强
  蓄势阶段: 粒子越来越密集 → 声音越来越"紧迫"

层3: 张力上升层（蓄势阶段叠加）
  类型: 谐波叠加 — 基频(300Hz) + 奇数倍泛音
  效果: 频率从300Hz缓慢滑向800Hz（跟随势能值）
  关键: 在临界前3秒加入 增四度(三全音) — 最不和谐的纯音程
  这替代了次声波的"压迫感" — 不和谐 > 低频 在笔记本上更有效

层4: 撕裂瞬间（一次性触发）
  时间线:
    0ms:     所有层静默
    300ms:   中高频冲击 — 带通噪声(800-4000Hz)突然爆发，持续200ms
             模拟"撕裂声" — 不是实际录制的声音，是合成纹理
    600ms:   泛音序列开始散开 — 多个正弦波以不同速度衰减
    1500ms:  过渡到层5
  关键: 0.3s静默是"屏息" — 制造期待。然后中高频爆发。
        笔记本扬声器的中高频响应好，这个爆发是有效的。

层5: 清明泛音层
  类型: 五声音阶泛音 — 宫商角徵羽的谐波序列
  基频: 523Hz (C5 = 宫)
  衰减: 每个音以不同的速度衰减（最快的0.5秒，最慢的15秒）
  效果: 像风铃被拨动后的余韵 — 清脆、逐渐散开
  持续: 约30-40秒后完全消失，只留呼吸层

混合:
  所有层通过Master Gain → 笔记本扬声器
  Master有软限幅防止削波
  默认音量: 70%（保留30%余量给瞬态爆发）
```

### 9.3 为什么这个声音设计在笔记本上行得通

| 传统做法（依赖低音炮/音响） | 笔记本替代方案 |
|--------------------------|--------------|
| 次声波制造压迫感 | 中频增四度音程制造紧张感 |
| 低频地震冲击 | 0.3s静默 + 中高频爆发（对比冲击） |
| 空间音频环绕感 | 左右声道节奏交替 + 随机粒子散布 |
| 巨大音量制造震撼 | 动态对比（巨大 vs 微小）制造震撼 |
| 持续低频嗡鸣 | 带通噪声的呼吸节奏 |

**核心洞察：笔记本扬声器不是"低配音响"——它是不同的乐器。设计声音时要把它当作一个"中频乐器"来对待。**
